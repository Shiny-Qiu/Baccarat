# Baccarat Simulator - PyQt6 百家乐模拟器

一款基于 PyQt6 的百家乐游戏模拟器，完整实现传统百家乐和免佣百家乐两套规则系统，支持 11 种投注玩法、可配置牌靴副数（1\~20 副）、逐张发牌动画、按牌靴自动分局的路单记录、逐局复盘系统和实时统计，让玩家身临其境地体验百家乐。

---

## 快速开始

### 环境要求
- Python 3.10+
- PyQt6

### 安装与运行
```bash
pip install PyQt6
python main.py
```

启动后弹出欢迎对话框，选择游戏模式、初始筹码（¥100 ~ ¥10,000,000）和牌靴副数（1\~20 副，默认 8 副），点击"开始游戏"进入主界面。

---

## 操作说明

| 操作 | 方式 |
|------|------|
| 选择牌靴副数 | 在开始对话框中设置（1\~20 副，默认 8 副） |
| 选择筹码面额 | 点击底部圆形筹码按钮（¥10 / 50 / 100 / 500 / 1K / 5K） |
| 下注 | **左键**点击投注区域，叠加当前面额的筹码 |
| 撤注 | **右键**点击投注区域，取回当前面额的筹码 |
| 发牌 | 点击 **"发牌"** 按钮，或按 **Enter / Space** |
| 不下注发牌 | 可以不下注直接点击 **"发牌"**，仅观看牌局走势 |
| 重复上局下注 | 点击 **"重复上局"** 按钮，一键复制上局的全部投注 |
| 清除全部下注 | 点击 **"清除下注"** 按钮，退还所有已押筹码 |
| 复盘记录 | 点击 **"复盘记录"** 按钮，查看本场所有局的下注与盈亏明细 |

## 游戏模式

### 传统百家乐
- 庄赢：1 赔 1，**抽取 5% 佣金**（向上取整）
- 闲赢：1 赔 1

### 免佣百家乐
- 庄赢：1 赔 1
- 庄以 **6 点**胜出时：赔率降为 **1 赔 0.5**
- 闲赢：1 赔 1

两种模式下，和局时庄/闲主注均**退还本金**（不赢不输）。

---

## 投注项目与赔率（共 11 种）

### 主注

| 投注项 | 赔率 | 胜出条件 |
|--------|------|----------|
| 庄 | 1:1（传统抽佣5%）/ 1:1 或 1:0.5（免佣） | 庄家点数 > 闲家点数 |
| 闲 | 1:1 | 闲家点数 > 庄家点数 |
| 和 | 1:8 | 双方点数相同 |

### 对子

| 投注项 | 赔率 | 胜出条件 |
|--------|------|----------|
| 庄对 | 1:11 | 庄家首两张牌**牌面相同**（如两张 K） |
| 闲对 | 1:11 | 闲家首两张牌**牌面相同** |

### 幸运 6 系列

| 投注项 | 赔率 | 胜出条件 |
|--------|------|----------|
| 幸运6 | 1:12（两牌）/ 1:20（三牌） | 庄以 6 点胜出，按庄家牌数自动匹配赔率 |
| 两牌幸运6 | 1:22 | 庄以**恰好两张牌** 6 点胜出 |
| 三牌幸运6 | 1:50 | 庄以**三张牌** 6 点胜出 |

### 幸运 7 系列

| 投注项 | 赔率 | 胜出条件 |
|--------|------|----------|
| 超级幸运7 | 1:30 / 1:40 / 1:100 | 闲以 7 点胜出且庄 6 点（按全桌总牌数 4/5/6 张） |
| 两牌闲幸7 | 1:6 | 闲**首两张牌** 7 点胜出 |
| 三牌闲幸7 | 1:15 | 闲**补牌后** 以 7 点胜出 |

> **超级幸运7 总牌数说明**：4 张 = 双方各 2 张（无补牌）；5 张 = 一方补了 1 张；6 张 = 双方各补了 1 张。

## 游戏流程

```
下注(可选) → 发首两牌 → 检查例牌(8/9点) → 搏牌(补第三张) → 比牌结算 → 记入复盘
```

1. **下注阶段** — 选筹码面额，左键点击投注区放置筹码，可同时押注多个位置；也可不下注直接发牌观看
2. **发牌阶段** — 逐张动画翻牌：闲1 → 庄1 → 闲2 → 庄2（每张间隔 0.5 秒）
3. **例牌检查** — 任一方首两牌合计 8 或 9 点（Natural），直接跳至结算
4. **搏牌阶段** — 先判断闲家是否补牌，再判断庄家（规则见下方详表），自动执行
5. **结算阶段** — 显示胜负结果、各投注项盈亏明细、更新筹码余额，自动写入复盘日志

### 补牌规则

#### 闲家

| 首两牌合计 | 动作 |
|-----------|------|
| 0 ~ 5 | 必须补牌 |
| 6 ~ 7 | 不补 |
| 8 ~ 9 | 例牌，不补 |

#### 庄家（闲家未补牌时）

| 首两牌合计 | 动作 |
|-----------|------|
| 0 ~ 5 | 必须补牌 |
| 6 ~ 7 | 不补 |

#### 庄家（闲家补了第三张牌时）

| 庄合计 | 闲第三张点数为以下时 **不补** | 其余情况 |
|--------|---------------------------|---------|
| 0 ~ 2 | —（无论如何都补） | 补牌 |
| 3 | 8 | 补牌 |
| 4 | 0, 1, 8, 9 | 补牌 |
| 5 | 0, 1, 2, 3, 8, 9 | 补牌 |
| 6 | 除 6 和 7 外均不补 | 仅闲第三牌为 6 或 7 时补 |
| 7 | —（一律不补） | 不补 |

> 10/J/Q/K 的点数均为 **0**，在庄家补牌判断中与 10 等价。

### 界面要素
- **牌面展示** — QPainter 自绘扑克牌（白底圆角矩形，左上角牌面值，中央花色符号，红/黑双色）
- **投注区域** — 深绿底金边方块，显示投注名/赔率/已押金额，悬停指针变手型，胜负时金色/红色高亮
- **筹码按钮** — 圆形按钮，仿真赌场配色（白/红/绿/蓝/黑/紫），选中时金色边框
- **发牌动画** — QTimer 驱动，每 0.5 秒翻一张牌，实时更新点数显示
- **路单记录** — 横向滚动区域，每局显示为独立结果块（编号、主结果、闲/庄点数、命中的边注标签）；牌靴用完重新洗牌后路单自动清空，从新牌靴第 1 局重新计数
- **统计面板** — 实时显示总局数、庄/闲/和胜率、庄对/闲对/幸运6/闲幸7 命中次数、当前连胜、牌靴剩余

## 牌靴配置

- 启动时可在开始对话框中选择牌靴副数（1\~20 副，默认 8 副）
- 每副 52 张标准扑克牌
- 牌靴剩余不足 52 张时自动重新洗牌，路单清空、开始新一副牌靴
- 顶部状态栏和统计面板实时显示牌靴剩余/总牌数
- 重新洗牌不影响筹码余额和下注状态，仅重置路单显示

---

## 路单记录

路单面板以横向滚动的方式展示**当前牌靴**的逐局结果。每局显示为一个独立的结果块：

| 信息 | 说明 |
|------|------|
| 局号 | 当前牌靴内编号 #1, #2, #3...（洗牌后重新从 #1 开始） |
| 主结果 | **庄**（红色）/ **闲**（蓝色）/ **和**（绿色） |
| 点数 | 闲家点数 : 庄家点数 |
| 命中标签 | 庄对、闲对、幸运6、两牌幸运6、三牌幸运6、两牌闲幸7、三牌闲幸7、超幸7·N牌 |

命中标签颜色编码：
- **对子**（庄对/闲对）：橙色
- **幸运6 系列**（幸运6/两牌幸运6/三牌幸运6）：金色
- **闲幸7 系列**（两牌闲幸7/三牌闲幸7）：浅蓝色
- **超级幸运7**：紫色

> 牌靴用完自动洗牌时，路单面板会清空并从新牌靴的 #1 重新开始，之前的记录仍完整保存在复盘系统中。

## 复盘记录

游戏内置完整的对局复盘系统，自动记录每一局的下注与盈亏情况，**按牌靴分组**展示，方便结束游戏后回顾分析整段游戏过程。

### 逐局记录内容

每局自动记录以下信息：
- **所属牌靴**编号
- **局号**与当前游戏模式
- **下注明细**：各投注项及金额（未下注则标记为"空注局"）
- **开牌结果**：闲/庄牌面（如 ♠A ♥5）、点数、胜者、是否例牌
- **命中标签**：庄对、闲对、幸运6、闲幸7 等
- **结算明细**：每项下注的赔付描述
- **本局盈亏**与**余额变化**（局前 → 局后）

### 牌靴分组

复盘窗口按牌靴分组显示所有对局记录：
- 每组标题显示 **"牌靴 #N"**、该牌靴的总局数和盈亏小计
- 最新牌靴排在最前，方便快速定位最近的对局
- 洗牌后的新一副牌靴自动归入新分组

### 本场总结

复盘窗口顶部显示整场游戏的汇总统计：
- 总局数、下注局数、空注局数、牌靴数
- 初始筹码与最终筹码
- 总下注额与总盈亏
- 最大单局盈利与最大单局亏损

### 查看方式
- **游戏中**：随时点击 **"复盘记录"** 按钮查看
- **游戏结束**：筹码用完时自动弹出复盘总结窗口，然后可选择重新开始或退出
- **重新开始**：选择重新开始后，旧会话记录自动清空，开始全新的复盘会话

---

## 筹码与结算

- **初始筹码**：¥100 ~ ¥10,000,000，在开始对话框设定
- **下注扣款**：每次点击投注区域立即从余额扣除对应面额
- **结算返还**：赢 → 退还本金 + 利润；输 → 无返还；和局退注 → 退还本金
- **佣金计算**（传统模式）：向上取整 `ceil(投注额 × 5%)`
- **游戏结束**：筹码不足最小面额（¥10）时自动弹出复盘总结，然后可选择重新开始或退出

## 项目结构

```
Baccarat/
├── main.py              # 程序入口，初始化 QApplication 和 Fusion 风格
├── card.py              # Card 类（花色、点数、颜色、显示符号）
├── shoe.py              # Shoe 类（可配置副数，自动洗牌）
├── game_logic.py        # BaccaratGame（发牌、补牌规则、洗牌检测、11 种投注结算、路单标签生成）
├── ui_main_window.py    # 全部 UI 组件（StartDialog / SessionReviewDialog / CardWidget / BetAreaWidget / MainWindow）
└── README.md
```

### 关键技术点
- **牌靴管理**：支持 1\~20 副标准扑克牌，剩余不足 52 张时自动重新洗牌，`RoundResult.new_shoe` 标记洗牌事件
- **牌靴分局**：洗牌后路单自动清空、编号重置，复盘记录按牌靴分组保存完整历史
- **补牌逻辑**：一次性预计算整局结果（所有牌），UI 仅负责逐步揭示动画
- **路单标签**：每局结束后由 `_build_road_tags()` 自动生成完整命中标签（庄对/闲对/幸运6/闲幸7/超幸7 等），确保与赔付规则一致
- **结算引擎**：统一的 `settle_bets()` 接口，支持传统/免佣双模式，返回每项投注的净盈亏和文字描述
- **状态管理**：发牌期间锁定全部下注控件，结算后自动解锁，上局投注可一键复制；支持不下注直接发牌观看
- **复盘系统**：每局结算时自动记录完整下注/盈亏/余额日志，按牌靴分组，游戏中随时可查，结束时自动弹出总结
